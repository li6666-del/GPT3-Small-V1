# GPT3-Small-V1 项目进程

更新时间：2026-05-12

## 1. 项目定位

本项目是一个从零训练的中英文 GPT 小模型实验。当前主线不是追求一次性做出通用大助手，而是先把 125M 基座训练出来，再用多轮 SFT 逐步补齐最基础的助手行为：

- 能识别用户是在提问或请求帮助。
- 能用中文回答中文问题，用英文回答英文问题。
- 能把一句话自然结束，不无意义续写网页、模板或训练语料残片。
- 能在简单数学、翻译、拒绝、计划、短解释等任务上保持格式稳定。
- 能在 greedy 和多个 seed 下都稳定，而不是只在某一个固定 seed 下看起来正常。

## 2. 工程能力

已经实现的核心模块：

- decoder-only TransformerLM。
- SwiGLU MLP。
- tied embedding/lm head。
- memmap token 数据集。
- bf16 AMP 训练。
- gradient accumulation。
- checkpoint 保存和 resume。
- JSONL 训练日志。
- cosine learning rate schedule。
- Hugging Face tokenizers 快速 BPE tokenizer 支持。
- Python tokenizer fallback。
- 文本生成脚本。
- SFT 数据读取和 mask 构建。
- assistant-only loss。
- SFT 训练内置 generation eval。

SFT trainer 当前支持：

- `messages` 格式。
- `prompt` / `response` 格式。
- `instruction` / `output` 格式。
- token ids 格式。
- `User/System/prompt` 部分 label 置为 `-100`。
- 只对 assistant response 和 EOT 计算 loss。
- generation eval 支持：
  - 固定 seed sample。
  - greedy `top_k=1`。
  - 多 seed 按 prompt 抽样。

## 4. 130k 基座生成评测

130k 基座已经能产生局部通顺的中英文文本，但本质仍是预训练语言模型，不是助手：

- 容易续写网页、表格、模板、新闻或训练语料残片。
- 中文问答经常漂到英文。
- 简单数学不可靠。
- 翻译和邮件任务会混入无关格式。
- 拒绝请求没有稳定边界。
- 停止能力弱，容易继续生成。

结论：

`step_130000.pt` 适合作为 SFT 起点，但不能直接作为对话助手使用。

## 5. SFT 实验总览

### V1: 50k synthetic bootstrap

路径：

```text
data/sft/v1
configs/sft_125m_v1.json
runs/sft-v1-50k-from-130k
```

设置：

- train 50k。
- valid 2k。
- learning rate 2e-5。
- max steps 2500。

结果：

- best valid loss 约 0.0151 @ step 1400。
- final valid loss 约 0.0194 @ step 2499。
- loss 很低，但生成失败。

暴露问题：

- 输出塌缩到 `A.A.A...`。
- 英文模板如 `Average...` 反复出现。
- 数学错误。
- 翻译、拒绝、邮件任务互相串扰。

结论：

V1 证明“只看 loss 很危险”。数据太模板化、学习率太激进，模型学到了表面格式，但没有稳定助手行为。

### V2: broad synthetic

路径：

```text
data/sft/v2
configs/sft_125m_v2.json
runs/sft-v2-broad-from-130k
```

设置：

- train 80k。
- valid 3k。
- learning rate 3e-6。
- max steps 600。

结果：

- final valid loss 约 0.0596。
- 中文概念解释和拒绝比 V1 好。
- 英文仍有重复。
- 数学、开放问答、邮件任务仍不稳定。

结论：

降低学习率后没有 V1 那么剧烈塌缩，但数据仍然不够真实，无法作为主线继续。

### V4.2: micro-overfit

路径：

```text

runs/sft-v42-micro-overfit-from-130k
```

设置：

- train 1,300。
- valid 130。
- learning rate 1e-5。
- max steps 180。
- 固定核心 prompt 和近似变体。

训练内表现：

- 从 step 45 起，训练内 generation eval 13/13 正确。
- final step 179 best loss 约 2.6382e-05。

独立加载 checkpoint 后暴露问题：

## 6. 当前主线判断

当前最有价值的路径是：

1. 不回到 V3 那种大规模真实混合数据。
2. 不急着追求 model style。
3. 继续以 V4.3 为锚点，小步扩展。
4. 每轮都必须通过 greedy 和多 seed generation eval。
5. 任何训练内看起来漂亮但独立加载后失败的结果，都不进入主线。

当前推荐继续点：

```text
runs/sft-v43-chinese-anchor-from-130k/step_000149.pt
```

## 8. 下一步建议：V4.4

V4.4 不应该大幅扩数据，而应该围绕 V4.3 已经稳定的能力做近邻扩展：

- 中文解释类 prompt 做 5 到 10 倍 paraphrase。
- 拒绝类加入更多危险请求表达，但输出保持短。
- 数学只扩到一位数和两位数基础加减。
- 翻译保持低比例，避免再次支配输出语言。
- 加入更多“请只回答一句话”“不要继续补充”的停止锚点。
- 每 25 或 50 step 做一次 generation eval。
- 评测必须同时包含 sample、greedy、多 seed。

V4.4 的目标不是通用性，而是让 V4.3 的稳定性从固定样本扩展到近邻样本。
