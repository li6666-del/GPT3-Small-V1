# GPT3-Small-V1 项目进程

更新时间：2026-05-14

## 1. 项目定位

本项目用于完整体验一个小型 LLM 的训练流程：从 tokenizer、预训练、SFT、评测到 checkpoint 管理。模型规模约 124M，目标是逐步得到一个能做简单问答和基础助手交互的小模型。

当前主线已经明确：停止回到 base model 做继续预训练，继续围绕 SFT 做小步稳定迭代。

## 2. 已完成内容

- 实现 124M decoder-only Transformer。

## 4. SFT 总结

早期实验：

- V1/V2：synthetic bootstrap，loss 很低但生成塌缩，证明不能只看 loss。
- V3：真实数据混合跨度太大，模型还没学好助手格式就被要求泛化，暂停。
- V4-V4.6：回到小步 SFT，重点修结束格式、中文短答、拒答、stop。
- V4.7-V4.15：逐步加入身份、能力说明、unknown、核心 QA/math 保护。
- V4.17：引入中文 held-out 和 pre-heldout gate。

## 5. V4.18 系列结论

V4.18 formal held-out：

- 没有通过正式 held-out。
- 身份、拒答、unknown、部分数学锚点较稳。
- stop exact、core QA、project terms、泛化数学仍弱。

V4.18.1-V4.18.8：

- 多轮尝试修 core QA、时间单位、项目术语和 hard residual。
- 结论：时间单位和 checkpoint/project terms 会明显污染其他能力，不适合继续作为主门槛强修。

V4.18.9：

- 目标：保存一个可用助手核心 checkpoint。
- 结果：通过 main gates，保留 `step_000055.pt`。
- 通过项：身份、非 OpenAI/ChatGPT、能力说明、拒答、stop、法国/北京/H2O、`1+4`、`5x8`。
- 未作为阻断项：时间单位、checkpoint 解释、部分称呼变体。

V4.18.10-V4.18.12：

- 目标：修 unknown 自然表达。
- 结果：均未保存 checkpoint。
- 原因：unknown 有局部改善，但会引入项目内部话术，或导致 `1 加 4` 退化成 `4 + 4 = 5`。
- 处理：本轮 `.pt` 已清理，只保留失败经验。

当前模型能做：

- 固定身份介绍。
- 简单能力说明。
- 基础拒答。
- 简短 stop 输出。
- 少量固定中文常识问答。
- 少量固定算术样本。

当前模型不能稳定做：

- 广泛中文问答。
- 泛化数学。
- 时间单位问答。
- 项目术语解释。
- 自然、通用的 unknown 边界表达。

## 7. 下一步建议

继续从 V4.18.9 出发，而不是从 V4.18.10-V4.18.12 出发。

推荐策略：

- 先守住身份、stop、拒答、`1+4`、`5x8`。
- unknown 只修一个高价值模板，不再同时修多个问法。
- 时间单位和 project terms 暂时作为 observe，不作为保存阻断。
- 每轮训练规模小，失败即删权重，避免污染主线。
